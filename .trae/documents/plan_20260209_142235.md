I will implement Kernel Version Management to allow users to switch between different versions of the `usque` kernel and view the version of the `official` kernel.

### Backend Implementation

1. **Create** **`KernelVersionManager`** **(`controller-app/app/kernel_version_manager.py`)**:

   * **Directory Structure**: Manage kernel binaries in a dedicated directory (e.g., `data/kernels/`).

     * `data/kernels/usque/v1.0.0/usque`

     * `data/kernels/usque/v1.1.0/usque`

   * **Methods**:

     * `list_versions(backend)`: Scan the directory for available versions.

     * `set_active_version(backend, version)`: Update configuration to use the selected version.

     * `get_active_version(backend)`: Return the currently configured version.

     * `get_binary_path(backend)`: Return the absolute path to the executable for the active version.
2. **Update** **`UsqueController`** **(`controller-app/app/usque_controller.py`)**:

   * Modify it to use `KernelVersionManager.get_binary_path('usque')` instead of calling `usque` directly from PATH.
3. **Update** **`OfficialController`** **(`controller-app/app/official_controller.py`)**:

   * Add method to fetch system installed version (e.g., `warp-cli --version`).

   * Mark as "System Managed" (read-only) in version selection.
4. **Add API Endpoints (`controller-app/app/main.py`)**:

   * `GET /api/kernel/versions`: Get list of available versions for the current backend.

   * `POST /api/kernel/version`: Switch to a specific version (triggers restart).

### Frontend Implementation

1. **Update** **`KernelManager.vue`** **(`frontend/src/views/KernelManager.vue`)**:

   * Add a **Kernel Version** card/section.

   * Display current version.

   * **Dropdown**: For `usque`, list available versions from the backend and allow switching.

   * **Read-only Display**: For `official`, show the installed version with a "System Managed" badge.

### Data Storage

* Kernel binaries will be stored in `controller-app/data/kernels/`.

* Configuration (active version) will be stored in `controller-app/data/config.json` or similar.

### Verification

* Verify that switching versions in UI updates the backend.

* Verify that `usque` starts with the correct binary path.
